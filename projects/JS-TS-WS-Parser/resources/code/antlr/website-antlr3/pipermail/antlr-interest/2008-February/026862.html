You're right, that wouldn't work in a parser with a:(X)=>Y;

X could be a prefix of Y.  So if these are lexer rules, then we would
have something like

A: (X)=>Y
   | ...
  ;

fragment X:...;

Y: X Z;

Where X is enough to disambiguate to choose the rule Y.  Likewise in a
parser it could be

a:(X)=>y
 | ...
;
