The problem is not in escaping '.'.
Do you have rules to skip whitespaces in Lexer?
You have to instruct ANTLR what to do with them.
Look in docs & example grammars for rules like this:

protected
NL : (   {LA(2) == '\n'}? '\r' '\n'
       | '\r'
       | '\n'
     )
     { newline(); }
   ;

WS : ( options { greedy = true; } :
          ' '
       | '\t'
       | '\f'
       | NL
     )+
     { _ttype = Token.SKIP; }
   ;
