>> parent: ^(parent_a B?) ;
>> parent_a: ^(PARENT A?) ;
>> --------------------------------------------
>

. . . is the same as

parent: ^( ^(PARENT A?) B? ) ;

which will not match ^(PARENT B), but will match ^( ^(PARENT) B).  BTW, 
a subtree match() will always properly consume the UP DOWN subtree 
delimiters.

You probably want

parent: ^(parent_a B?) ;
parent_a: PARENT A? ;
