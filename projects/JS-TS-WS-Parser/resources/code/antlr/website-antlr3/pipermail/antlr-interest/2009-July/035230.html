hakan eryargi wrote:
> thank you but i can't get it. simply making AND's and OR's have same
> precedence doesnt force user to use parentheses.

Try this:

  tokens {
    OR     = '|';
    AND    = '&';
    LPAREN = '(';
    RPAREN = ')';
  }

  expr
    : primaryExpr ((OR^ primaryExpr)+ | (AND^ primaryExpr)+)?
    ;

  primaryExpr
    : LPAREN expr^ RPAREN
    | simpleExpr
    ;
