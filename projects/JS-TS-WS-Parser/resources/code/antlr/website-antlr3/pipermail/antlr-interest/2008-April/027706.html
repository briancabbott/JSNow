David,

I think all you need to add to your grammar is a list of allowed types
as separate tokens like:
type_specifier
	: 'void'
	| 'char'
	| 'short'
	| 'int'
	| 'long'
	| 'float'
	| 'double'
	| 'signed'
	| 'unsigned';

This way there would be no ambiguity (I think)
