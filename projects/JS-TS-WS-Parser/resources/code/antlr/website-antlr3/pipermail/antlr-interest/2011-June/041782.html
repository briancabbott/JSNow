I have an issue with antlrworks (1.4.2), where for a specific grammar,
it requires a whitespace.

        | '%' string ':' percentage=INT
        | ...

LOWID    : ('a'..'z'|'\-')+;
INT :   ('0'..'9')+ ;

If I call my example rules with:
  ?\"acgt\"
it works fine
but if I call
 %\"acgt\":30

If fails.

At least if I add a whitespace between % and \"acgt\", it works:

here....
