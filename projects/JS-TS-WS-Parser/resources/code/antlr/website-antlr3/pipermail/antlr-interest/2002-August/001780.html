I should add that antlr is trying to protect itself from creating a variable
name that won't work in the target language.  It constructs the name from
your prefix "LITERAL_" and then the keyword.  

First: does this cause a problem for you, or are you just curious?
Second:  I believe you can use the tokens section to add a keyword and a
token name for it.  


>>From http://www.antlr.org/doc/metalang.html#_bb1:

You can also define literals in this section and, most importantly, assign
to them a valid label as in the following example.

tokens {
    KEYWORD_VOID="void";
    EXPR;
    DECL;
    INT="int";
}
