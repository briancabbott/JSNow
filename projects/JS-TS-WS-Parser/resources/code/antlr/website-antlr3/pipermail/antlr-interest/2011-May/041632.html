Hi Ben,

You cannot use fragment rules inside parser rules. So yes, you need to make
LKU a normal token. If you don't want that (for whatever reason), you need
to include the '&' in the 'NAME' rule:

NAME
  :  '&' ('a'..'z')+
  ;


Regards,

Bart.
