Mr. Ashford :-

Try:

-----------------------------------
tokens {
   RESULT="result";
}

protected
DIGIT  : ('0'..'9');

protected
ALPHA  : ('a'..'z' | 'A'..'Z');

ID
options {
  paraphrase   = "an identifier";
  testLiterals = true;
}
       : ALPHA (ALPHA | DIGIT)*;
-----------------------------------

That is, place your keywords into a tokens{} block and not in a lexer rule.
(I think the tokens{} block has to appear before any rules, but can't
remember for sure right now...)

Hope this helps...
