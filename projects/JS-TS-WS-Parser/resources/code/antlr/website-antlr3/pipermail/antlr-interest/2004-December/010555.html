> > > Why does the following simple grammar create an ambiguous

> > > 
> > > 
> > > class MyLexer extends Lexer;
> > > 
> > > options {
> > > 	charVocabulary = '\3'..'\377';
> > > 	k=3;
> > > 	caseSensitive=false;
> > > }
> > > 
> > > COMMENT_COMMAND
> > >   : '-' '-' '$'
> > >   ;
> > >   
> > > SQL_STATEMENT
> > >   : ( SYMBOL )+ ';'
> > >   ;
> > > 
> > > protected SYMBOL
> > >   : ('-' (~ '-')) => '-'   //1
> > >   | 'a'
> > >   | '"' (~ '"')* '"'       //2
> > >   ;
> > > 
> > > When I remove either of //1 or //2, the ambiguity goes away. 
> > > When I replace //2 with 
> > >
