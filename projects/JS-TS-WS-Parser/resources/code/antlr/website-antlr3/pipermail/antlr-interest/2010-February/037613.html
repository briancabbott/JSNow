Hi Michael,

I have great news for you.  ANTLR does this for you.  Here's an example:

//
// Parser rules

statement: pos | neg ;

pos: ID 'is' ID ;

neg: 'not' ID ;

//
// Lexer rules

ID: ('A'..'Z' | 'a'..'z')+ ;

This example works because ANTLR tests tokens produced by ID against 
literal values used in the parser.  Try it out.

Cheers
