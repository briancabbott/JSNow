I'm building a small application that uses StringTemplate, and I find I need to
do on-the-fly aggregate creation. StringTemplate supports that capability, but
I need to be able to create aggregates with more than five values. I can't
pre-create the aggregate, because the contents of the aggregate depend on a
file passed in by the user. (The application is written in Scala, so it uses
the Java StringTemplate API, thus the restriction on a maximum of five
parameters when creating an aggregate.)

The restriction on five elements in an on-the-fly aggregate is problematic. If
the setAttribute() method taking an Object array were public, I could simply
call that. However, it is protected. Morever, given that StringTemplate
instances are created via factory methods in StringTemplateGroup, it does not
appear feasible to extend StringTemplate with a subclass.

Is there a reason the setAttribute() method taking Object[] is protected, or
can it be made public? If it must be protected, is there another solution to my
problem (short of generating bytecode on the fly, which is doable, but a last
resort)?
